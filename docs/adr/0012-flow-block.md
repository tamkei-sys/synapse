# ADR-0012: 埋め込み型フロー（ノードグラフ）ブロック — 自前SVGレンダラ + ノード属性格納

- 状態: 採択 (PRJ-16 / PBI-131〜135, 137)
- 日付: 2026-06-04
- 関係 PBI: PRJ-16（埋め込み型フロー・ビジュアライザ）/ PBI-131〜137
- 関連: packages/blocks/src/flow.ts / apps/web/src/features/flow/\* / apps/web/src/features/editor/flow-node.tsx / slash-menu.tsx / editor.tsx

## 背景

「AI で作った大規模システムの全体構造を、ノードグラフで描き、実行順にステップ
再生して見失わずに把握する」── という可視化（参考: 拡散した X 投稿の
"条件付き確率 生成パイプライン" 動画）を、SYNAPSE のドキュメント内に埋め込み
ブロックとして実装する。要件は色分けノード + エッジ + パン/ズーム + 凡例 +
ミニマップ + 実行順ステップ再生（STEP インジケータ）+ ノード詳細パネル。

論点は (1) フロー定義をどこに格納するか、(2) グラフ描画に何を使うか
（ランタイム依存の追加は CLAUDE.md 上 ADR 必須）、(3) スキーマの置き場所。

## 選択肢

### 1. フロー定義の永続表現

1. **専用 DB `flow` Block 型 + tRPC で別保存**（sheet 埋め込み方式）
   - ✅ 巨大グラフや共有・再利用に強い
   - ❌ ルータ・マイグレーション・保存配線が必要で重い。手で書く自己完結図には過剰
2. **専用 `flowBlock` atom ノード + `doc`(JSON文字列)属性（採用）**
   - ✅ `mermaidBlock` / `mathBlock` と完全に同型 — atom + 文字列属性 +
     ReactNodeView。永続化は Yjs ドキュメントに乗って**自動**、保存呼び出し不要
   - ✅ コピペ/履歴/共同編集が既存機構にそのまま乗る
   - ❌ 1図のサイズは属性に収まる範囲（スキーマで上限）。超巨大化したら選択肢1へ移行

### 2. グラフ描画ライブラリ

1. **React Flow (@xyflow) / cytoscape / d3 等を追加**
   - ✅ パン/ズーム/ミニマップが既製
   - ❌ ランタイム依存が増える（CLAUDE.md は強く抑制）。STEP 再生・詳細パネル・
     凡例・あの暗いステージ意匠はどのみち自作で、ライブラリの取り分が小さい
2. **依存ゼロの自前 SVG/HTML レンダラ（採用）**
   - ✅ 新規ランタイム依存なし（`pnpm install` 不要、コンテナ依存の地雷回避）
   - ✅ 単一の CSS transform で world→screen を写像。エッジは SVG ベジェ、ノードは
     絶対配置 HTML（リッチな内容を素直に描ける）。意匠を完全制御
   - ❌ 自動レイアウト（dagre 等）は無し。ノード座標は手置き（"手で定義する図"なので可）。
     将来オートレイアウトが要れば純関数として後付けする

### 3. スキーマの置き場所

- **`packages/blocks/src/flow.ts` に Zod で `FlowDoc` を定義（採用）**: カテゴリ/
  ノード/エッジ/ステップ + 参照整合（`superRefine`）。トポロジヘルパ（Kahn 層分け、
  ステップ自動導出）も純データ層として同居。UI 側のジオメトリ（ノード寸法・ベジェ）
  は `features/flow/geometry.ts` に分離する。

## 決定

- 選択肢 **1-2 / 2-2 / 3** を採用。`flowBlock` atom ノード、自前 SVG レンダラ、
  スキーマは `packages/blocks`。**新規ランタイム依存は追加しない。**
- `/flow` スラッシュはサンプル（条件付き確率 生成パイプライン）を挿入して即再生可能に。
  「▶ 一括実行」は `steps` を順に点灯。`steps` 未定義の図はトポロジ層から自動導出する。
- ステージは**テーマ非依存の暗い面**（メディアプレイヤー同様の可視化サーフェス）。
  カテゴリ色はデータ駆動なので inline style、それ以外の chrome は zinc/violet トークン。
- ビジュアル編集 UI は PBI-136 に分離。それまでの繋ぎとして NodeView に
  **生 JSON エディタ**を備え、エンドツーエンドで編集可能にしておく（不正 JSON は
  インラインエラーにフォールバックしソースを保持）。

## 制約・帰結

- 公開ページ（read-only）は対象外。`apps/api/.../public-doc.ts` の ALLOWED_NODES に
  `flowBlock` を**まだ追加しない** ── 公開時はサニタイザがノードを落とす（graceful）。
  公開対応が必要になれば、許可リスト追加 + 公開エディタ登録を別 PBI で行う。
- エッジは左→右フロー前提のベジェ。逆向き（バックエッジ）は描けるが見た目が最適でない。
  本格的な経路最適化・オートレイアウトは後続。
- 大規模グラフ（数百ノード）はスキーマ上限まで属性に載るが、その規模では選択肢1
  （専用 Block + 仮想化）への移行を検討する。
